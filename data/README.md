# Data

This directory contains the raw and processed data used in the NYC street vendor enforcement analysis. Raw data is not committed to version control (see `.gitignore`); this README documents provenance and how to reproduce each dataset locally.

## Directory structure

```
data/
  raw/
    shapefiles/
      council_districts/
      bid_boundaries/
    council_districts.geojson
    acs_2022.csv
    311_requests.csv
    legistar_votes.csv
  processed/
    model_df.parquet
    district_scores.parquet
```

## Sources

| Dataset | Source | Access method | Notes |
|---|---|---|---|
| Council District boundaries | NYC Open Data — City Council Districts | Socrata API (GeoJSON) | Confirm redistricting vintage (pre/post-2023) matches other layers |
| BID boundaries | NYC Dept. of Small Business Services | Direct download (shapefile) | Year-aware; BID membership assigned per model year |
| 311 Service Requests | NYC Open Data | Socrata API | Filtered to relevant complaint types and date range |
| ACS 5-Year Estimates | U.S. Census Bureau | `census` Python API | Vars: B01003, B19013, etc. — see `src/data/acs.py` |
| Pro-vendor legislative votes | NYC Council Legistar API | REST API | 1998–2025 floor votes, 14 enacted bills |

## Reproducing the raw data

Each dataset can be regenerated by running `notebooks/01_data_collection.ipynb`, or by calling the individual fetch functions in `src/data/`:

```python
from src.data import fetch_council_districts, fetch_bid_boundaries, fetch_311, fetch_acs, fetch_legistar_votes

fetch_council_districts(out="data/raw/council_districts.geojson")
fetch_bid_boundaries(out="data/raw/shapefiles/bid_boundaries/")
fetch_311(out="data/raw/311_requests.csv")
fetch_acs(out="data/raw/acs_2022.csv")
fetch_legistar_votes(out="data/raw/legistar_votes.csv")
```

### Council District boundaries

```bash
curl -o data/raw/council_districts.geojson \
  "https://data.cityofnewyork.us/resource/yusd-j4xi.geojson?$limit=51"
```

> Verify `yusd-j4xi` is still the current Socrata resource ID before running — NYC occasionally republishes datasets under a new ID after redistricting cycles.

## Coordinate reference system

All spatial layers are projected to **EPSG:2263** (NAD83 / New York Long Island, feet) to match `model_df`.

## Processed data

- `model_df.parquet`: council-district-level (n=51) panel with outcome `n_violations`, offset `total_pop`, and predictors including `pct_in_bid_z`, `311_requests_z`, `median_dist_km_z`.
- `district_scores.parquet`: pro-vendor legislative scoring by district, derived from Legistar floor votes.

Processed files are built from raw data via `notebooks/02` through `notebooks/05` — see each notebook's header for the specific transformation steps.
